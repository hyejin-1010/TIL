# 알고리즘

## 1) 검색 알고리즘

### 선형 검색
	* 배열의 인덱스를 처음부터 끝까지 하나씩 증가시키면서 방문하여 그 값이 속하는 지를 검사

```
For i from 0 to n-1
	If i'th element is 50
		Return true
Return fasle
```

### 이진 검색
	* 배열이 정렬되어 있는 경우, 배열 중간 인덱스부터 시작하여 찾고자 하는 값과 비교하여 그보다 작은 인덱스 또는 큰 인덱스로 이동을 반복한다.

```
If no items
	Return false
If middle item is 50
	Return true
Else if 50 < middle item
	Search left half
Else if 50 > middle item
	Search right half
```

## 2) 알고리즘 표기법
### Big O 표기법
	* 실행 시간의 상한
	* O는 ‘on the order of’의 약자로, ‘~만큼의 정도로 커지는 것’이라고 볼 수 있다.
	* O(n/2)도 결국 n이 커지면 1/2은 큰 의미가 없어지므로, O(n)이라고 볼 수 있다.

> - O(n^2)  
> - O(n log n)  
> - O(n) : 선형 검색  
> - O(log n) : 이진 검색  
> - O(1)  

### Big Ω
	* 알고리즘 실행 시간의 하한

> Ω(n^2)  
> Ω(n log n)  
> Ω(n) : 배열 안에 존재하는 값의 개수 세기  
> Ω(log n)  
> Ω(1) : 선형 검색, 이진 검색
